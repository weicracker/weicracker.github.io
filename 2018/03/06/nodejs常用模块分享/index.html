nodejs常用模块分享

发表于 2018-03-06 | 分类于 node

简介

以下为本人在工作开发过程中常用的node模块及其简单介绍，以备后续工作中方便查找与使用

常用模块

日志管理 ： 「Log4js」

  • 简介

    类似于JAVA中的log4j，具有coloured console logging、可以设置输出等级等功能，简单易用，如果没有复杂需求足够使用了。winston是一个功能更强大也更复杂的日志模块，人气比较高，对功能有追求的可以看看这个模块。

  • 简单使用

http调用 ： 「request」「axios」

  • 简介

    可以理解为一个简化的CURL模块，建立各种HTTP请求，支持授权验证，其实功能还是非常多的，基本满足绝大部分需求。目前在NPM模块排行前几名。

  • 简单使用

简单部署 ： 「forever」

  • 简介

    forever可以看做是一个nodejs的守护进程，能够启动，停止，重启我们的app应用，运行稳定，支持高并发，启动/停止命令简单，支持热部署，宕机重启，监控界面和日志，集群环境。pm2是一个带有负载均衡功能的Node应用的进程管理器。当你要把你的独立代码利用全部的服务器上的所有CPU，并保证进程永远都活着，0秒的重载。

  • 简单使用

    实际部署 ： 「pm2」

  • 简介

    pm2是非常优秀工具,它提供对基于node.js的项目运行托管服务。它基于命令行界面,提供很多特性: 内置的负载均衡器等等

  • 简单使用

    定时任务 ：「node-schedule」

  • 简介

    node-schedule是一个小巧的定时任务管理模块，可以满足我们绝大部分需求

  • 简单使用

    日期工具 : 「moment」

  • 简介

    moment是一个日期工具库，主要提供了对日期的解析，验证，操作和格式化等，如果需要对日期作较多的操作或比较复杂的操作，则它是一个不错的选择，毕竟JS自带的Date功能太弱了

  • 简单使用

    数据操作库 : 「lodash」

  • 简介
  • 简单使用

    数据唯一身份创建 ： 「uuid」

  • 简介
  • 简单使用

    本地文件操作 : 「fs-extra」

  • 简介

    fs-extra模块是系统fs模块的扩展，提供了更多便利的 API，并继承了fs模块的 API

  • 简单使用

    数据压缩 ： 「pako」

  • 简介

    pako解析压缩字符串问题

  • 简单使用

    文件后缀名处理库 ： 「mime」

  • 简介

    MIME(Multipurpose Internet Mail Extensions)多用途互联网邮件扩展类型。是设定某种扩展名的文件用一种应用程序来打开的方式类型，当该扩展名文件被访问的时候，浏览器会自动使用指定应用程序来打开。多用于指定一些客户端自定义的文件名，以及一些媒体文件打开方式
